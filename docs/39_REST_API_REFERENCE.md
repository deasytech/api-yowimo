# Claude Code Instructions

When implementing APIs:

1. Keep controllers thin.
2. Delegate business logic to services.
3. Validate all input with Form Requests.
4. Use API Resources for responses.
5. Return standardized JSON.
6. Never expose internal exceptions.
7. Document every endpoint before release.
8. Update this document whenever APIs are added or modified.

---

# Acceptance Criteria

The REST API is complete when:

- Every endpoint is documented.
- Authentication is standardized.
- Responses are consistent.
- Validation and error handling follow a common format.
- Pagination, filtering, and sorting are supported.
- Financial endpoints are idempotent.
- The API can evolve through versioning without breaking existing clients.
